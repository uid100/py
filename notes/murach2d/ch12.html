A dictionary in Python is a collection of key-value pairs. Each key is unique and associated with a value.

    Dictionaries are
  • mutable,
  • unordered,
  • and can contain any data type as values.

Mutable refers to the ability of an object to be changed after it is created. In the context of dictionaries, being mutable means that you can modify the contents of the dictionary after it has been created. This includes adding new key-value pairs, updating existing values associated with keys, and removing key-value pairs

 

In Python, dictionaries are considered unordered data structures. This means that the elements in a dictionary do not have a defined order. Unlike sequences such as lists or tuples, where elements are stored in a specific order and accessed by index, the items in a dictionary are stored and retrieved by their keys, and their order is not guaranteed. When you iterate over a dictionary or print its contents, you might observe that the items are not necessarily displayed in the same order in which they were inserted.

 

Creating Dictionaries

In Python, dictionaries are created using curly braces {}. You can initialize an empty dictionary or populate it with key-value pairs:


# Creating an empty dictionary
my_dict = {}

# Creating a dictionary with initial key-value pairs
my_dict = {'key1': 'value1', 'key2': 'value2'}

Accessing, Setting, Adding, and Deleting Items

  • Accessing: You can access the value associated with a key using square brackets [] or the get() method.
  • Setting: To set or update the value associated with a key, simply assign a new value to that key.
  • Adding: To add a new key-value pair, simply assign a value to a new key.
  • Deleting: You can remove a key-value pair using the del keyword or the pop() method.

Looping Through Keys and Values

You can iterate through a dictionary's keys, values, or key-value pairs using loops or methods like keys(), values(), and items():


# Looping through keys
for key in my_dict:
    print(key)

# Looping through values
for value in my_dict.values():
    print(value)

# Looping through key-value pairs
for key, value in my_dict.items():
    print(key, value)

Converting Between Dictionaries and Lists

You can convert dictionaries to lists using methods like list() and dict() to convert lists to dictionaries. When converting a dictionary to a list, you can choose to convert keys, values, or key-value pairs.

 

Differences Between Dictionaries and Lists

While both dictionaries and lists are used to store collections of data, they have different characteristics:

  • Dictionaries use key-value pairs for data storage, while lists use ordered sequences of elements.
  • Dictionaries are unordered, so the order of items is not guaranteed, whereas lists maintain the order of elements.
  • Dictionaries are preferred when data needs to be accessed quickly using unique keys, while lists are suitable for maintaining an ordered collection of items.

Dictionaries vs JSON

In a previous module we looked at JSON-formatted data files, key-value pairs which looked very similar to this, but with some important differences.

Comparison: Python Dictionaries vs. JSON

Syntax

Python dictionaries are defined using curly braces {} and colons : to separate keys and values. JSON uses a similar syntax but with double quotes " around keys and values.

Data Types

Python dictionaries can contain any data type as keys or values, including integers, strings, lists, tuples, other dictionaries, etc. JSON supports a subset of the data types available in Python, including strings, numbers, booleans, arrays (lists in Python), and objects (equivalent to Python dictionaries).

Usage

Python dictionaries are a built-in data type used within Python programs to store and manipulate data. JSON is a lightweight data interchange format used for transmitting data between a server and a web application, or between different systems. It is language-independent and widely used for representing structured data.

Serialization and Deserialization

Python dictionaries can be easily converted to JSON strings using the json.dumps() function from the json module, and JSON strings can be parsed into Python dictionaries using json.loads(). This process is known as serialization (converting Python objects to JSON) and deserialization (converting JSON back to Python objects).

Ordering

Python dictionaries are unordered, meaning they do not preserve the order of elements. JSON objects, on the other hand, are ordered collections of key-value pairs, meaning the order of elements is preserved when converting between JSON and Python dictionaries.
